# Московская биржа
Сервис справочника данных по ценным бумага Московской биржи, 
а также истории сделок по этим ценным бумагам.

###Запуск сервиса

Для установки и работы сервиса необходимо:
  - база данных PostgreSQL;
  - Версия JRE 11.0.8.
  
Конфигурационный файл базы данных расположен src/main/resources/db_config.txt, в данном файле 
необходимо прописать в формате JSON данные для работы сервиса - имя рабочей базы данных, имя пользователя и пароль.
   ```json
{
     "host":"localhost",
     "port":8000
}        
   ```

Конфигурационный файл для запуска сервера расположен src/main/resources/server_config.txt, в данном файле 
необходимо прописать в формате JSON данные для работы сервиса - хост и порт.
   ```json
{
     "host":"localhost",
     "port":8000
}
   ```

###Описание API

API позволяет:

Получать информацию о ценных бумагах и сделок по ним в виде html-таблицы. 

1. Получить стартовую html-страницу, отправив GET запрос на http://"host":"port".

2. Получить html-страницу для выполнения операций с информацией по ценным бумагам, отправив 
GET запрос на http://"host":"port"/security.

3. Получить информацию по ценной бумаге, отправив  GET запрос
 на http://"host":"port"/security/read  с обязательным параметром
    - secid - тикер ценной бумаги для поиска в базе.
    
4. Удалить информацию о ценной бумаге,  отправив  GET запрос
на http://"host":"port"/security/delete  с обязательным параметром
   - secid - тикер ценной бумаги для поиска в базе.
   
5. Получить список ценных бумаг, отправив GET запрос на http://"host":"port"/security/list с 
обязательными параметрами:
    - limit - количество выводимых элементов;
    - offset - смещение от начала списка;
а также с необязательным параметром:
    - sort_parameter - параметр сортировке при выводе списка (secid или emitent_title").
    
6. Получить страницу для формирования запроса на добавление бумаги в базу данных, отправив 
GET запрос на http://"host":"port"/security/add.

7. Добавить информацию о ценной бумаге в базу данных, отправив POST запрос на 
 http://"host":"port"/security/add с данными по бумаге в формате JSON:
    ```json
        {
           "secid":"AAPL",
           "shortname":"Apple",
           "regnumber":" 111",
           "name":" ",
           "isin":"US0378331005",
           "is_traded":"1",
           "emitent_id":"1281003",
           "emitent_title":"Apple Inc",
           "emitent_inn":"1212",
           "emitent_okpo":"1212",
           "gosreg":"121",
           "type":"common_share",
           "group":"stock_shares",
           "primary_boardid":"EQRD\t",
           "marketprice_boardid":"EQRD",
           "history":[]
        }
    ```
При ручном сохранении ценной бумаги   передаваемые данные в поле name могут быть - 
только кириллица, цифры и пробел.

8. Получить страницу для обновления информации о ценной бумаге - отправить 
GET запрос на http://"host":"port"/security/update с параметром:
    - secid - тикер обновляемой ценной бумаги.
    
9. Обновить информацию о ценной бумаге в базе данных, отправив POST запрос на 
http://"host":"port"/security/update с новыми данными по бумаге в формате JSON:
   ```json
   {
      "secid":"AAPL",
      "shortname":"Apple",
      "regnumber":" 111",
   ```

При ручном сохранении ценной бумаги   передаваемые данные в поле name могут быть -

10. Получить страницу для добавления информации о ценных бумагах из файла 
выгрузки Московской биржи (securities_*.xml) - отправить GET запрос 
на http://"host":"port"/security/add/all.

11. Добавить информацию о бумагах из файла выгрузки Московской биржи (securities_*.xml),
отправить POST запрос на http://"host":"port"/security/add/all c
переданными данными из файла.

на http://"host":"port"/security/history  с обязательным параметром
- secid - тикер ценной бумаги для поиска в базе.

13. Получить html-страницу для выполнения операций с информацией по сделкам с ценными бумагами, 
отправив GET запрос на http://"host":"port"/transaction.
    
14. Получить информацию о сделке по ценной бумаге, отправив  GET запрос
на http://"host":"port"/transaction/read  с обязательным параметром
    - id - id операции в базе.

        - filter_parameter_value - значение для параметра фильтрации.


### Описание архитектуры приложения

HTTP-сервер ExchangeHttpServer принимает запросы по REST API и передает их в репозиторий Repository.
Репозиторий прокидывает запрос в ExchangeDataSource. 
Репозиторий позволяет использовать множество источников данных, 
но на текущий момент реализован один - база данных.
ExchangeDataSource реализует подключение к базе данных PostgreSQL.
Работа с базой данных реализована с использованием Hibernate ORM.
